Java中的IO流

Posted on 2015-05-28   |   In java   |  

按数据传输单位分类：

  • 字节流：以字节(8位二进制)为单位进行处理。主要用于读写诸如图像或者声音的二进制数据。字节流继承InputStream和OutputStream。
  • 字符流：以字符(16位二进制)位单位进行处理。字符流继承自InputStreamReader和OutputStreamWriter。

字符流和字节流有什么区别？

都是通过字节流的方式实现。字符流是对字节流进行了封装，方便操作。在底层中，所有的输入输出都是字节形式的。当字符向字节转换时，要注意编码问题，因为字符串转成字节数组。

什么是Java序列化，如何实现Java序列化？

Java提供了一种对象序列化的机制，该机制中，一个对象可以被表示为一个字节序列，该字节学列包括对象的数据，有关对象的类型的信息和存储在对象中数据的类型。将序列化对象写入文件后，可以从文件读取出来，然后对其进行反序列化，也就是说，对象的类型信息，对象的数据，还有对象中的数据类型可以用来在内存中新建对象。这整个过程是JVM独立的，也就是说在一个平台上序列化的对象，可以在另一个平台进行反序列化。

在Java中，一个类实现了java.io.Serializable接口，那么它就可以被序列化。然后我们可以调用OutputStream的writeObject方法对对象进行保存，然后用readObject方法对保存的对象进行读取。
